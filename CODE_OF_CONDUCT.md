# Código de Conducta

Este proyecto adopta un entorno de colaboración profesional, respetuoso y técnicamente orientado a la excelencia.

La gobernanza técnica definida en este repositorio requiere un entorno de trabajo basado en respeto, responsabilidad y claridad.

## 1. Propósito

Este Código de Conducta tiene como objetivo:

- Fomentar una cultura de colaboración profesional.
- Garantizar discusiones técnicas constructivas.
- Proteger la integridad del proyecto y de sus colaboradores.
- Establecer expectativas claras de comportamiento.

Aplica a todos los espacios del proyecto, incluyendo:

- *Pull Requests*
- *Issues*
- Revisiones de código
- Discusiones técnicas
- Documentación

## 2. Comportamiento Esperado

Se espera que todos los participantes:

- Mantengan un trato respetuoso y profesional.
- Fundamenten sus argumentos en criterios técnicos.
- Acepten revisiones y retroalimentación constructiva.
- Eviten confrontaciones personales.
- Mantengan foco en la calidad técnica y la gobernanza.
- Colaboren de forma transparente.

Las decisiones deben priorizar:

- Integridad estructural
- Determinismo
- Seguridad
- Coherencia con la gobernanza definida

## 3. Comportamiento Inaceptable

No se tolera:

- Ataques personales.
- Lenguaje ofensivo o despectivo.
- Conducta intimidatoria.
- Sabotaje intencional del flujo de trabajo.
- Intentos deliberados de debilitar controles de gobernanza.
- Introducción maliciosa de vulnerabilidades o secretos.

Las conductas que afecten la integridad técnica del proyecto también pueden considerarse incumplimiento del código.

## 4. Resolución de Conflictos

En caso de desacuerdo técnico:

1. Argumentar con base en evidencia.
2. Referenciar documentación normativa.
3. Priorizar el contrato estructural del *baseline*.
4. Escalar a un mantenedor si no se alcanza consenso.

Las decisiones finales deben alinearse con:

- `docs/governance-baseline.md`
- `docs/versioning.md`
- `docs/security-model.md`

La gobernanza técnica prevalece sobre preferencias individuales.

---

## 5. Responsabilidad de Mantenedores

Los mantenedores deben:

- Aplicar este código de forma consistente.
- Actuar con imparcialidad.
- Priorizar la integridad del proyecto.
- Intervenir ante comportamientos inapropiados.
- Garantizar que la discusión técnica permanezca profesional.

## 6. Aplicación

El incumplimiento de este Código de Conducta puede resultar en:

- Advertencia formal.
- Restricción temporal de participación.
- Reversión de cambios.
- Revocación de privilegios de contribución.
- Exclusión del proyecto, en casos graves.

La aplicación debe ser proporcional y documentada.

## 7. Declaración Final

Este repositorio promueve una cultura de excelencia técnica.

La gobernanza estructural requiere disciplina, respeto y profesionalismo.

Al participar en este proyecto, aceptas cumplir con este Código de Conducta.
